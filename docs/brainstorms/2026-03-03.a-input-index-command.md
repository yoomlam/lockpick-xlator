# Brainstorm: `/index-inputs` Slash Command

**Date:** 2026-03-03

---

## What We're Building

A new slash command `/index-inputs` that scans a domain's `input/policy_docs/` directory and produces an auto-generated `core/input-index.yaml` file. The index contains one entry per H1–H3 heading section across all `.md` files, with file path, heading text (prefixed with `#`/`##`/`###` to encode level), a 1-sentence summary, and 3-5 topic tags — all LLM-generated by reading the full file content.

The index is the **first place to look** when searching for which document covers a given policy topic. It reduces time spent grepping through raw policy docs by providing a structured, scannable overview. Specific documents can then be read for full detail.

---

## Key Decisions

| Decision | Choice | Rationale |
|---|---|---|
| File location | `domains/<domain>/specs/input-index.yaml` | Co-located with other manifests (`extraction-manifest.yaml`, `naming-manifest.yaml`) that commands reference |
| Content per entry | file path + heading (with `#` prefix) + summary + tags | Per-section granularity; markdown prefix encodes level without a separate field |
| Update behavior | Fully regenerate each run | Keeps index in sync with files; consistent with `extraction-manifest.yaml`; header says `do not edit manually` |
| Path format | Domain-relative (e.g., `input/policy_docs/apa_manual/441/441-1 EARNED INCOME.md`) | Matches format used in `extraction-manifest.yaml` |

## Output Format

```yaml
# Auto-generated by /index-inputs — do not edit manually
# Domain: ak_doh
# Generated: 2026-03-03
sections:
  - path: input/policy_docs/apa_manual/441/441-1 EARNED INCOME.md
    heading: "# 441-1 EARNED INCOME"
    summary: "Defines earned income types counted toward APA eligibility, including wages, tips, self-employment, and FICA contributions."
    tags: [earned income, wages, tips, self-employment, FICA]

  - path: input/policy_docs/apa_manual/441/441-1 EARNED INCOME.md
    heading: "## Types of Earned Income"
    summary: "Lists specific wage and salary categories that constitute earned income under APA rules."
    tags: [wages, salary, tips, FICA, net earnings]

  - path: input/policy_docs/apa_manual/442/442-2 EARNED INCOME EXCLUSIONS.md
    heading: "# 442-2 EARNED INCOME EXCLUSIONS"
    summary: "Specifies amounts excluded from earned income before APA benefit calculation."
    tags: [exclusions, earned income, deductions, earned income disregard, APA]
```

---

## Command Interface

```
/index-inputs <domain>
```

- Single argument: domain name (e.g., `ak_doh`, `snap`)
- Prints error if `domains/<domain>/input/policy_docs/` does not exist
- Always regenerates `domains/<domain>/specs/input-index.yaml` from scratch

---

## Command Steps

1. Validate `<domain>` exists and has `input/policy_docs/`
2. Glob all `*.md` files recursively under `input/policy_docs/`
3. For each file: read full content, extract H1–H3 headings with their section text, generate 1-sentence summary + 3-5 tags per section; files with no headings get a single entry using the filename as heading
4. Sort entries by path (alphabetical), then by heading appearance order
5. Write `core/input-index.yaml` with auto-generated header

---

## How Other Commands Will Use the Index

- **`/extract-ruleset`**: Read `core/input-index.yaml` first to identify which docs are relevant to the program being extracted, then read only those files in full. Reduces context window usage.
