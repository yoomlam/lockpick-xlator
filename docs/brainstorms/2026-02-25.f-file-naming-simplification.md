# File Naming Simplification: Remove Redundant Suffix

**Date:** 2026-02-25
**Status:** Decided — ready to implement

---

## What We're Changing

Remove the `-brainstorm` suffix from files in `docs/brainstorms/` and the `-plan` suffix from files in `docs/plans/`. The directory name already communicates file type; the suffix is redundant.

**Before:** `docs/brainstorms/2026-02-25.a-extract-ruleset-determinism-brainstorm.md`
**After:** `docs/brainstorms/2026-02-25.a-extract-ruleset-determinism.md`

**Before:** `docs/plans/2026-02-25.c-feat-pydantic-civil-schema-plan.md`
**After:** `docs/plans/2026-02-25.c-feat-pydantic-civil-schema.md`

---

## Why This Approach

- **No tooling relies on the suffix.** No scripts, commands, or skills glob or reference these files by suffix pattern.
- **Directory name is sufficient context.** Claude reads file contents, not just filenames; placement in `docs/brainstorms/` vs `docs/plans/` disambiguates file type.
- **Cleaner filenames.** Shorter, less noisy, consistent with standard project file naming conventions.
