# Brainstorm: Replace Makefile with Python CLI

**Date:** 2026-03-03
**Status:** Ready for planning

---

## What We're Building

Replace the current copy-paste-per-domain Makefile entirely with:
1. A Python script `x.py` as the **core** ‚Äî argparse CLI with `rich` output
2. A thin shell shim `./x` ‚Äî users never type `python`

Adding a new domain requires **zero changes** to any existing files ‚Äî just create the directory structure and CIVIL spec.

The `textual` TUI is out of scope for MVP; deferred to a follow-on.

---

## Why This Approach

The current Makefile requires copying a ~20-line block per domain (5 config vars + 5-6 targets). Python is chosen over shell for:
- **Cross-platform compatibility** ‚Äî no bash dependency
- **OPA lifecycle management** ‚Äî `./x test` can start/stop OPA as a subprocess
- **`rich` library** ‚Äî clean colored output, tables for `./x list`
- **Extensibility** ‚Äî easier to add discovery, validation, error messages

A thin `./x` shell shim keeps the entry point ergonomic (no `python` prefix). The Makefile is deleted entirely.

---

## Interface Design

### Entry point: `./x` shell shim

```sh
#!/bin/sh
exec python x.py "$@"
```

Must be `chmod +x ./x`.

### Actions

```bash
# Project setup
./x setup                             # install uv, venv, OPA

# Domain/module pipeline
./x validate  snap eligibility        # validate CIVIL YAML
./x transpile snap eligibility        # generate Rego
./x test      snap eligibility        # start OPA, run tests, stop OPA
./x demo      snap eligibility        # start OPA + FastAPI demo
./x graph     snap eligibility        # generate computation graph
./x pipeline  snap eligibility        # validate ‚Üí transpile ‚Üí test (stops on first failure)

# Utilities
./x generate-schema                   # regenerate core/ruleset.schema.json
./x list                              # discover all domain/module pairs
```

`./x list` output example:
```
snap         eligibility
ak_doh       apa_adltc
ak_doh       adltc_2024
ak_doh       adltc_2025
ak_doh       adltc
```

## Key Decisions

- **Python + `rich`** ‚Äî CLI with colored output; `textual` TUI deferred
- **`./x` shim** ‚Äî ergonomic entry point; `chmod +x`
- **Action first**: `./x <action> <domain> <module>`
- **OPA lifecycle**: `test` and `pipeline` start OPA as subprocess, run tests, then stop it (self-contained for CI)
- **`pipeline` stops on failure** ‚Äî mirrors Makefile dependency chain (transpile depends on validate)
- **`./x list`** ‚Äî discovers all `domains/*/specs/*.civil.yaml` (all files including drafts)
- **`./x setup`** ‚Äî replaces `make baseline-setup` (uv, venv, OPA install)
- **`./x generate-schema`** ‚Äî replaces `make generate-schema`
- **Makefile deleted** ‚Äî no Makefile at all; `./x` is the sole entry point

## Paths Derived by Convention

All paths derived from `domain` and `module` arguments:

| Variable | Path |
|---|---|
| CIVIL | `domains/<domain>/specs/<module>.civil.yaml` |
| REGO | `domains/<domain>/output/<module>.rego` |
| TESTS | `domains/<domain>/specs/tests/<module>_tests.yaml` |
| PACKAGE | `<domain>.<module>` |
| OPA_PATH | `/v1/data/<domain>/<module>/decision` |
| DEMO_SCRIPT | `domains/<domain>/demo/start.sh` |

These conventions match existing domains (`snap/eligibility`, `ak_doh/apa_adltc`) exactly.

## Resolved Questions

- **Stay with Make or replace?** ‚Üí Python script (`x.py`); Makefile deleted
- **Shell or Python?** ‚Üí Python ‚Äî OPA subprocess management, `rich` output, cross-platform
- **TUI (`textual`)?** ‚Üí Deferred to follow-on; MVP is CLI only
- **Invocation style?** ‚Üí Action first (`./x validate snap eligibility`)
- **Multiple modules per domain?** ‚Üí Yes, both `domain` and `module` are explicit args always
- **`./x setup` and `./x generate-schema`?** ‚Üí Yes, both implemented as `./x` actions
- **OPA in `test`/`pipeline`?** ‚Üí Start automatically as subprocess; self-contained for CI
- **Discovery scope in `./x list`?** ‚Üí All `*.civil.yaml` files in `domains/*/specs/`
- **`pipeline` failure behavior?** ‚Üí Stop on first failure (validate ‚Üí transpile ‚Üí test)
