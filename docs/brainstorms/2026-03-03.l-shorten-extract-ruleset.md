# Shorten `extract-ruleset.md` for Reduced Token Cost

**Date:** 2026-03-03
**Status:** Draft

---

## What We're Building

Reduce the token cost of loading `.claude/commands/extract-ruleset.md` on each invocation.
The file is 842 lines. Every run loads the entire file regardless of which mode is needed.

---

## Why This Approach

Goal is token cost reduction per invocation, not maintainability.
The two biggest wins are structural (split modes) and content consolidation (collapse CIVIL Reference).
These are independent and can be done together.

---

## Key Decisions

### 1. Split into two commands

Create two separate command files:

| Command | File | Mode |
|---------|------|------|
| `/extract-ruleset` | `.claude/commands/extract-ruleset.md` | CREATE only |
| `/update-ruleset` | `.claude/commands/update-ruleset.md` | UPDATE only |

- **Mode Detection section** (lines 99–111) is eliminated — each command IS its mode
- Pre-flight, Scoring Rubric, CIVIL Reference, and Shared Procedures are **duplicated** in both files (not shared externally — external file reads cost tokens too)
- Current 842-line file → ~530 lines per CREATE invocation, ~415 lines per UPDATE invocation (before further optimizations from Decisions 2–3)

Estimated savings from split alone: **37% for CREATE runs, 51% for UPDATE runs**

### 2. Replace CIVIL Reference with a compact external constraints file

The 35-line inline CIVIL Reference section contains two categories:
- Schema constraints and gotchas (already duplicated in `core/civil-quickref.md`)
- Two items NOT yet in civil-quickref.md: Expression Language syntax and `source:` vs `citations:` distinction

**Why not just link to civil-quickref.md:** It's 206 lines. A "Read civil-quickref.md now" directive
trades 35 inline lines for 206 loaded lines — net negative.

**Plan:** Create `core/civil-authoring-constraints.md` (~20 lines) containing only:
- Expression language operators/functions (the items not in civil-quickref.md)
- `source:` vs `citations:` distinction
- A brief pointer to civil-quickref.md for full attribute tables

Replace the 35-line inline section with a directive to read this compact file.
Net saving: ~15 lines per command file (35 removed, 20 loaded).

### 3. Compress verbose print messages

Pre-flight currently quotes exact terminal output verbatim (e.g., the full "No AI guidance found"
block). Replace with intent descriptions: "Print an error naming the missing file and suggest
the correct command." Claude generates equally good output without exact templates.

Estimated savings: **~20 lines per file**

### 4. Keep the YAML template in Step 4

The user confirmed: the YAML template is generic scaffolding, not a domain-specific example.
It stays in the CREATE command.

---

## Resolved Questions

- **Use a shared include file?** No. External file reads cost tokens (Claude loads both files). Duplication across two self-contained command files is cheaper.
- **Replace YAML template with link to snap example?** No. The snap domain may not exist in other deployments.
- **CIVIL Reference: link to civil-quickref.md?** No — it's 206 lines, net negative. Create compact `core/civil-authoring-constraints.md` (~20 lines) instead.
- **Update command name:** `/update-ruleset`
- **Guard in `/extract-ruleset`:** If a CIVIL file already exists, print "A CIVIL file already exists for this domain. Did you mean `/update-ruleset`?" and stop. Replaces the Mode Detection section.
