# Brainstorm: Rule Preview Step in /refine-guidance

**Date:** 2026-03-03
**Status:** Draft

---

## What We're Building

A preview gate at the end of Step 4 (Q&A refinement) in `/refine-guidance`. After all four guidance sections have been refined through Q&A, the AI presents 2–3 sample rules it would extract — paired with the source policy text they came from — and asks the user to confirm or refine before writing `ai-guidance.yaml`.

This closes a feedback gap: currently, the user refines abstract guidance (constraints, standards, etc.) but can't verify how that guidance would shape actual rule extraction until they run `/extract-ruleset`. The preview makes the connection tangible.

---

## Why This Approach

**Re-use Step 3 observations.** Step 3 already analyzes the policy documents to generate doc-derived proposals for each guidance section. Those observations (policy excerpts, identified rule patterns) are still in context when Step 4 completes — the preview can be assembled from this existing material without re-reading any documents. This keeps the command fast and avoids extra LLM reads.

**Targeted back-navigation.** If a rule looks wrong, the user picks which section caused the problem (e.g., "edge_cases") and that section's Q&A question is re-asked with the current value pre-filled. This is surgical: no need to redo all four sections.

**Unlimited confirmation loop.** The loop continues until the user explicitly approves. This fits the stakes: `ai-guidance.yaml` is a prerequisite for every future `/extract-ruleset` run on this domain, so it must be right.

---

## Key Decisions

| Decision | Choice | Rationale |
|---|---|---|
| When does preview appear? | After all 4 Q&A sections complete | Showing mid-flow (after each section) would be disruptive and premature |
| How are preview rules generated? | Re-use Step 3 observations (no re-read) | Avoids latency; Step 3 data is already in context |
| How many sample rules? | 2–3 | Enough to validate, not so many the user is overwhelmed |
| What does each preview entry show? | CIVIL rule snippet + quoted source policy text | Grounds abstract rule in concrete policy language |
| Back-navigation scope? | Re-ask just the chosen section's Q&A, pre-filled | Surgical — avoids re-running all 4 sections |
| How many preview/refine rounds? | Unlimited — loop until user confirms | Stakes are high; user must feel confident before writing |
| What triggers write? | Explicit user confirmation ("looks good") | Never write `ai-guidance.yaml` until user approves |

---

## The Flow

```
Step 4 Q&A completes (all 4 sections)
          ↓
  [Preview Gate]
  AI presents:
    • 2-3 sample rules (CIVIL snippet + source text)
    • "Do these look right?"

  User options:
    A) "Looks good" → write ai-guidance.yaml → done
    B) "Refine section X" → re-ask that section's Q&A (pre-filled)
                         → return to Preview Gate
```

The loop between Preview Gate and section Q&A continues until the user selects option A.

## What the Preview Looks Like (Sketch)

```

Preview: Rules this guidance would extract

Rule 1 — gross income test (categorical)
  Source: "Households must have gross income at or below 130% of the
           Federal Poverty Level..."
  CIVIL:
    rules:
      - id: gross_income_eligible
        when: gross_income <= gross_limit
        then: passes_gross_income_test = true

Rule 2 — shelter deduction cap
  Source: "The shelter deduction is capped except for households with
           elderly or disabled members..."
  CIVIL:
    computed:
      - name: shelter_deduction
        conditional:
          - if: has_elderly_or_disabled
            then: shelter_expense - 0.5 * adjusted_net_income
          - else: min(shelter_expense - 0.5 * adjusted_net_income, shelter_deduction_cap)
─────────────────────────────────────────────
Do these look right?
  [a] Yes, write ai-guidance.yaml
  [1] Refine constraints
  [2] Refine standards
  [3] Refine guidance_notes
  [4] Refine edge_cases
  [x] Start over (discard all refinements)
```

## Scope

**In scope:**
- Preview gate at end of Step 4 in `/refine-guidance`
- 2–3 sample rules with source text, drawn from Step 3 observations
- Numbered menu offering section-level back-navigation
- Unlimited preview/refine loop
- Write `ai-guidance.yaml` only on explicit approval

**Out of scope:**
- Showing a full draft extraction (that's `/extract-ruleset`)
- Per-section preview (too disruptive mid-flow)
- Free-form YAML editing (guided Q&A is the interaction model)
- Running `/extract-ruleset` automatically from this command
