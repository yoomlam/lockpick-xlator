# feat: Computation Graph Tool

## Overview

Add a `computation-graph` capability that reads a CIVIL DSL YAML file and produces two artifacts
alongside it: a structured dependency graph in YAML (`<ruleset>.graph.yaml`) and a Mermaid
visualization (`<ruleset>.graph.md`). The graph documents every node in the dataflow — input
facts, computed intermediates, and rule sinks — with `depends_on`/`used_by` links.

The feature comprises four deliverables:
1. `tools/civil_expr.py` — shared AST-based expression reference extraction
2. `tools/computation_graph.py` — CLI tool that produces the graph files
3. Makefile `<domain>-graph` targets per domain
4. `/extract-ruleset` integration — auto-generates graph as the final step

## Motivation

CIVIL rulesets with many computed fields (SNAP has 12) are hard to reason about without a
visual dataflow map. A computation graph makes the dependency chain explicit, supports debugging,
and will enable downstream tools (e.g., impact analysis when a rule changes).

## Technical Approach

### Shared Expression Parser — `tools/civil_expr.py`

CIVIL expressions are valid Python syntax with one known exception: the `in()` function
(e.g., `in(value, [a, b, c])`) uses `in` as a name, which is a Python reserved keyword.
Pre-parse, replace `in(` with `in_(` to make it parseable; treat `in_` as a known function name
to filter from refs. All other CIVIL operators (`&&`, `||`, `!`) are valid Python operators.

```python
# tools/civil_expr.py

import ast
import re
from dataclasses import dataclass, field

# Function names in CIVIL expressions that are not references
_CIVIL_FUNCTIONS = {"max", "min", "exists", "is_null", "between", "in_", "table"}

# Pre-parse fix: 'in' is a Python keyword; rename to 'in_' before parsing
_IN_FN_RE = re.compile(r'\bin\(')

@dataclass
class ExprRefs:
    entity_fields: list[str] = field(default_factory=list)  # "Entity.field"
    computed_refs: list[str] = field(default_factory=list)  # bare names in computed set
    constant_refs: list[str] = field(default_factory=list)  # UPPER_SNAKE names not in tables
    table_refs:    list[str] = field(default_factory=list)  # table('name', ...) calls

def extract_refs(
    expr: str,
    computed_names: set[str],
    table_names: set[str],
) -> ExprRefs:
    """Walk the AST of a CIVIL expression and return categorized references.

    Uses a collected set of attribute-value node ids to suppress entity names
    (e.g. 'Household') from appearing as bare Name references.
    """
    expr = _IN_FN_RE.sub("in_(", expr)
    tree = ast.parse(expr, mode="eval")
    refs = ExprRefs()

    # First pass: collect all node.value.id from Attribute nodes and
    # all table name strings so we can suppress them in the Name pass.
    attribute_value_ids: set[str] = set()
    table_call_args: set[str] = set()
    for node in ast.walk(tree):
        if isinstance(node, ast.Attribute):
            # Only suppress if value is a simple Name (PascalCase entity names)
            if isinstance(node.value, ast.Name):
                attribute_value_ids.add(node.value.id)

    # Second pass: collect refs
    for node in ast.walk(tree):
        if isinstance(node, ast.Attribute):
            # Guard: only record Attribute if value is a bare Name (Entity.field pattern).
            # Excludes table('name', key).column — node.value is a Call, not a Name.
            if isinstance(node.value, ast.Name):
                refs.entity_fields.append(f"{node.value.id}.{node.attr}")
        elif isinstance(node, ast.Call):
            func = node.func
            if isinstance(func, ast.Name) and func.id == "table" and node.args:
                try:
                    table_name = ast.literal_eval(node.args[0])
                    refs.table_refs.append(table_name)
                except (ValueError, TypeError):
                    pass
        elif isinstance(node, ast.Name):
            name = node.id
            # Skip entity names (already captured as part of Attribute), function names
            if name in attribute_value_ids or name in _CIVIL_FUNCTIONS:
                continue
            if name in computed_names:
                refs.computed_refs.append(name)
            elif name in table_names:
                refs.table_refs.append(name)  # bare table name reference
            elif name == name.upper() and len(name) > 1:
                refs.constant_refs.append(name)

    return refs

def extract_refs_from_computed(
    field,  # ComputedField-like object with .expr / .conditional
    computed_names: set[str],
    table_names: set[str],
) -> ExprRefs:
    """Extract refs from a computed field, handling both expr and conditional forms."""
    if field.get("expr"):
        return extract_refs(field["expr"], computed_names, table_names)
    else:
        cond = field["conditional"]
        r1 = extract_refs(cond["if"],   computed_names, table_names)
        r2 = extract_refs(cond["then"], computed_names, table_names)
        r3 = extract_refs(cond["else"], computed_names, table_names)
        return ExprRefs(
            entity_fields=r1.entity_fields + r2.entity_fields + r3.entity_fields,
            computed_refs=r1.computed_refs  + r2.computed_refs  + r3.computed_refs,
            constant_refs=r1.constant_refs  + r2.constant_refs  + r3.constant_refs,
            table_refs=   r1.table_refs     + r2.table_refs     + r3.table_refs,
        )
```

**Notes on correctness:**
- `ast.walk` visits all nodes including children. The two-pass approach collects
  entity names (from Attribute `.value.id`) in the first pass, then suppresses
  them as bare `ast.Name` refs in the second pass.
- `table('name', key).column` — the outer `ast.Attribute` node has `node.value` as a
  `Call`, not a `Name`. The `isinstance(node.value, ast.Name)` guard prevents the crash
  and skips the column-access attribute.
- Only `rule.when` is traversed for rule dependency extraction. `then:` actions are side
  effects (not data dependencies) and are excluded by design.

### Node Key Convention

The `nodes` dict in the output YAML uses these keys:

| Kind | Key | Example |
|------|-----|---------|
| Fact field | `"Entity.field_name"` | `"Household.earned_income"` |
| Constant | Constant name (as-is) | `"EARNED_INCOME_DEDUCTION_RATE"` |
| Table | Table name (as-is) | `"standard_deductions"` |
| Computed | Field name (as-is) | `"net_income"` |
| Rule | Rule id (as-is) | `"FED-SNAP-DENY-001"` |

`depends_on` and `used_by` lists use these same keys. The `used_by` reverse index is built
by iterating all nodes and appending the current node's key to each of its dependency's
`used_by` list.

**Allow rules:** All rules (both `deny` and `allow`) are included as nodes. Allow rules with
`when: "true"` will have empty `depends_on` — this is correct (they are unconditional sinks).

**Constant type:** Use Python `isinstance`: `int → "int"`, `float → "float"`,
`str → "string"`, `bool → "bool"`. No `money` inference — constants don't carry currency
context in the CIVIL DSL.

### Graph Tool — `tools/computation_graph.py`

Follows the `validate_civil.py` boilerplate: shebang, module docstring with Usage/Exit codes,
`fail()`, `load_civil()`, `main()` guard. Loads raw YAML (not Pydantic) to match the
existing tool pattern.

**Algorithm:**

```
1. parse args: positional <civil_yaml>
2. load_civil(path) → raw dict
3. Derive output paths:
     out_dir   = pathlib.Path(path).parent          # same dir as input
     stem      = pathlib.Path(path).stem            # e.g. "eligibility.civil"
     program   = stem.replace(".civil", "")         # "eligibility"
     domain    = out_dir.parent.parent.name         # "snap" (from domains/snap/specs/)
     graph_yaml = out_dir / f"{program}.graph.yaml"
     graph_md   = out_dir / f"{program}.graph.md"

4. Collect known name sets:
     computed_names = set(doc.get("computed", {}).keys())
     table_names    = set(doc.get("tables", {}).keys())
     constant_names = set(doc.get("constants", {}).keys())

5. Build nodes dict (insertion order = topological order):

   [INPUTS — fact fields]
   For each entity_name, entity_def in doc["facts"].items():
     For each field_name, field_def in entity_def["fields"].items():
       key = f"{entity_name}.{field_name}"
       nodes[key] = {kind: "input", type: field_def["type"],
                     location: f"facts.{entity_name}.{field_name}",
                     depends_on: [], used_by: []}

   [INPUTS — constants]
   For each name, value in doc.get("constants", {}).items():
     nodes[name] = {kind: "input", type: python_type(value),
                    location: f"constants.{name}",
                    depends_on: [], used_by: []}

   [INPUTS — tables]
   For each name in doc.get("tables", {}).keys():
     nodes[name] = {kind: "input", type: "table",
                    location: f"tables.{name}",
                    depends_on: [], used_by: []}

   [COMPUTED — in declaration order]
   For each field_name, field_def in doc.get("computed", {}).items():
     nodes[field_name] = {kind: "computed", type: field_def["type"],
                          location: f"computed.{field_name}",
                          description: field_def.get("description"),
                          depends_on: [], used_by: []}

   [RULES]
   For each rule in doc["rules"]:
     nodes[rule["id"]] = {kind: "rule", rule_kind: rule["kind"],
                          location: f"rules[{rule['id']}]",
                          depends_on: [], used_by: []}

6. Build depends_on for computed + rule nodes:
   For each computed field_name:
     refs = extract_refs_from_computed(field_def, computed_names, table_names)
     depends_on = dedupe([
       *[f"{e}" for e in refs.entity_fields],   # "Entity.field" — matches node keys
       *refs.computed_refs,
       *refs.constant_refs,
       *refs.table_refs,
     ])
     nodes[field_name]["depends_on"] = depends_on

   For each rule:
     if rule["when"] == "true":
       depends_on = []
     else:
       refs = extract_refs(rule["when"], computed_names, table_names)
       depends_on = dedupe([entity_fields, computed_refs, constant_refs, table_refs])
     nodes[rule["id"]]["depends_on"] = depends_on

7. Build used_by (reverse index):
   For each node_key, node in nodes.items():
     For each dep in node["depends_on"]:
       nodes[dep]["used_by"].append(node_key)

8. Serialize nodes to YAML (preserve insertion order; use yaml.dump with sort_keys=False):
   {
     domain: <domain>,
     program: <program>,
     generated: str(datetime.date.today()),
     nodes: {<key>: <node_dict>, ...}
   }
   Write to graph_yaml

9. Generate Mermaid:
   Sanitize node IDs: replace non-alphanumeric with "_"
   Check for collisions; append "_N" suffix if collision
   Build "graph TD\n" then for each node:
     inputs:   node_id(["key\ntype"])
     computed: node_id["key\ntype"]
     rules:    node_id{{"key"}}
   Then for each edge dep → node:
     dep_id --> node_id
   Wrap in ```mermaid ... ``` fenced block
   Write to graph_md

10. Print success lines:
    ✓ Graph written: <graph_yaml>
    ✓ Diagram:       <graph_md>
```

## Dependencies & Risks

- **Risk**: `ast.parse()` fails on a CIVIL expression. Mitigation: the `in(` → `in_(` pre-pass
  handles the only known reserved-keyword case. Other SyntaxErrors should be surfaced as `fail()`
  with the offending expression shown.
- **Risk**: Mermaid node ID sanitization produces collisions. Mitigation: detect collisions
  after sanitization and append `_2`, `_3` etc.
- **Out of scope**: Refactoring `transpile_to_opa.py` to import from `civil_expr.py`. The
  transpiler uses regex *substitution* (not just identification) — a different concern. Excluded
  to minimize risk.
