# feat: Add `/create-demo` Slash Command

## Overview

Add a `.claude/commands/create-demo.md` slash command that generates a working FastAPI + browser demo for any domain's policy module. The command reads the domain's CIVIL spec and test manifest to produce four tailored files in `domains/<domain>/output/demo-<module>/`.

## Problem Statement

Every new domain needs a demo app to make policy rules interactive and tangible. Currently only SNAP has one (`domains/snap/output/demo-eligibility/`), and it was written by hand. Replicating it for a new domain requires knowing the FastAPI pattern, Pydantic model derivation rules, OPA path conventions, and the HTML form structure — all of which can be automated.

## Proposed Solution

One new file: `.claude/commands/create-demo.md`. No template files, no new tooling. The slash command's `.md` instructs Claude to:

1. Resolve domain/module from optional args or by scanning `domains/` for CIVIL specs
2. Run pre-flight checks
3. Detect CREATE vs UPDATE mode
4. Read the CIVIL spec + test manifest
5. Generate the 4 demo files using the mappings below

### CIVIL Spec → Generated File Mappings

| CIVIL field | Generated artifact |
|-------------|-------------------|
| `facts.<Entity>.fields` | `HouseholdFacts` Pydantic model fields in `main.py`; `<input>` elements in `index.html` |
| `computed:` keys | `ComputedBreakdown` Pydantic model fields in `main.py`; deduction breakdown table rows in `index.html` |
| `decisions:` keys | `EligibilityResponse` Pydantic model fields in `main.py`; results section in `index.html` |
| `metadata.domain` + module name | OPA path `/v1/data/<domain>/<module>/decision`; FastAPI route `/api/<domain>/<module>` |
| First 3 test cases from `<module>_tests.yaml` | JS `EXAMPLES` dict in `index.html` (pick cases with distinct outcomes: allow, deny, edge) |
| No test file present | Placeholder `EXAMPLES` with `TODO` values; print warning: "No test manifest found. Run `/create-tests <domain>` first for realistic example scenarios." |

### Field Type Mappings (CIVIL → HTML form)

| CIVIL type | HTML input |
|-----------|------------|
| `bool` | `<input type="checkbox">` |
| `int` | `<input type="number" step="1">` |
| `money` | `<input type="number" step="0.01">` |
| `string` | `<input type="text">` |
| `optional: true` | Not required; default from CIVIL `default:` or 0/false |

### Generated Files

```
domains/<domain>/output/demo-<module>/
├── requirements.txt        # static boilerplate (fastapi, uvicorn[standard], httpx, pydantic)
├── start.sh               # launcher: start OPA on :8181, FastAPI on :8000
├── main.py                # FastAPI app with Pydantic models derived from CIVIL spec
└── static/
    └── index.html         # browser form derived from CIVIL facts:, 2-3 example scenarios
```

## Command Design

### Input Signature

```
/create-demo [<domain>] [<domain> <module>]
```

- **No args**: scan `domains/*/specs/*.civil.yaml`, prompt with numbered list if multiple found
- **One arg** (`<domain>`): use that domain, auto-detect module from its CIVIL file(s)
- **Two args** (`<domain> <module>`): skip scanning entirely

### Pre-flight Checks (in order)

1. Domain folder exists? — NO → `"Domain '<domain>' not found. Run /new-domain <domain> first."` Stop.
2. CIVIL file exists at `domains/<domain>/specs/<module>.civil.yaml`? — NO → `"No CIVIL file found. Run /extract-ruleset <domain> first."` Stop.
3. Rego file exists at `domains/<domain>/output/<module>.rego`? — NO → `"No Rego found. Run /transpile-and-test <domain> <module> first."` Stop.

### Mode Detection

```bash
ls domains/<domain>/output/demo-<module>/ 2>/dev/null
```

| Result | Mode |
|--------|------|
| Directory absent | CREATE mode |
| Directory present | UPDATE mode — prompt: "Demo already exists at `domains/<domain>/output/demo-<module>/`. Regenerate and overwrite? [y/N]" — abort on N |

### Process — CREATE Mode

**Step 1: Read inputs**
- Load `domains/<domain>/specs/<module>.civil.yaml`
- Load `domains/<domain>/specs/tests/<module>_tests.yaml` if present

**Step 2: Generate `requirements.txt`**
Static content (no domain-specific content):
```
fastapi
uvicorn[standard]
httpx
pydantic
```

**Step 3: Generate `start.sh`**
Follow the SNAP `start.sh` pattern exactly:
- `REPO_ROOT` anchored to script location (`$(cd "$(dirname "$0")/../../.."; pwd)`)
- `REGO_FILE="$REPO_ROOT/domains/<domain>/output/<module>.rego"`
- Prerequisite check for Rego with fix instruction: `"./x transpile <domain> <module>"`
- OPA started in background on port 8181, PID captured for cleanup trap
- FastAPI launched with `cd "$SCRIPT_DIR" && uvicorn main:app --host 0.0.0.0 --port 8000 --reload`
- Cleanup trap kills OPA on exit

**Step 4: Generate `main.py`**
Follow the SNAP `main.py` pattern:
- Constants: `OPA_URL = "http://localhost:8181"`, `OPA_DECISION_PATH = "/v1/data/<domain>/<module>/decision"`
- `InputFacts` Pydantic model: one field per `facts.<Entity>.fields` entry with type, description, and optionality from CIVIL
- `ComputedBreakdown` Pydantic model: one `float` field per `computed:` key
- `DenialReason` model: `{ code: str, message: str }` (standard)
- `DecisionResponse` model: fields from `decisions:` + `computed: ComputedBreakdown`
- API route: `@app.post("/api/<domain>/<module>", response_model=DecisionResponse)`
- POST body: `{"input": facts.model_dump()}`
- Unpack OPA result into `DecisionResponse`

**Step 5: Generate `static/index.html`**
Follow the SNAP `index.html` pattern:
- Form fields: one `<div class="form-group">` per `facts.<Entity>.fields` entry; type determines `<input>` kind
- `EXAMPLES` JS dict: 2-3 entries from test cases (pick `allow_*`, `deny_*`, edge case); use `TODO` placeholders with warning if no test file
- `renderResults()` JS function: iterate `computed.*` fields by name for breakdown table; render `denial_reasons[]` list; set eligible/denied badge
- `fetch('/api/<domain>/<module>', ...)` endpoint matches `main.py`

**Step 6: Print next steps**
```
Demo created at domains/<domain>/output/demo-<module>/

Next steps:
  1. Run the demo:  ./x demo <domain> <module>
  2. Open browser:  http://localhost:8000/static/index.html
  3. API docs:      http://localhost:8000/docs
```

### Process — UPDATE Mode

Same as CREATE mode (after user confirms overwrite). Overwrite all 4 files.

### Common Mistakes to Avoid

- Do NOT hardcode `snap` or `eligibility` anywhere — derive all names from `<domain>` and `<module>` args
- Do NOT include Rego generation logic — Rego is a pre-existing prerequisite
- Use `facts.<Entity>.fields` keys verbatim as Python attribute names (they are already snake_case)
- The `computed:` block may have `conditional:` entries — still extract the key name, not the expression
- The `decisions:` block may have a `list` type field (`denial_reasons`) — map to `list[DenialReason]`

## Technical Considerations

### Path Formulas (from `x.py` `resolve_paths()`)

All paths follow the convention already established in `x.py`:
- CIVIL spec: `domains/<domain>/specs/<module>.civil.yaml`
- Rego: `domains/<domain>/output/<module>.rego`
- Demo dir: `domains/<domain>/output/demo-<module>/`
- OPA package: `<domain>.<module>`
- OPA decision path: `/v1/data/<domain>/<module>/decision`
