# refactor: Move demo dir into output/demo-\<module>

Move `domains/<domain>/demo/` → `domains/<domain>/output/demo-<module>/` so all generated and runtime artifacts for a module live under `output/`.

## Motivation

The `demo/` directory is module-specific (it serves one OPA module's decision endpoint), so co-locating it under `output/` with the generated `.rego` file makes the layout more consistent. Adding a second module to a domain would produce a collision at `demo/`; `output/demo-<module>` avoids that.

## Acceptance Criteria

- [ ] `domains/snap/demo/` renamed to `domains/snap/output/demo-eligibility/` via `git mv`
- [ ] `./x demo snap eligibility` still starts OPA + FastAPI correctly
- [ ] `start.sh` REPO_ROOT resolution is correct from the new location (4 levels up, not 3)
- [ ] `x.py resolve_paths()` returns new paths for `demo_sh` and `demo_req`
- [ ] `x.py cmd_demo()` error message shows the new suggested path
- [ ] `/new-domain` scaffold creates `output/demo-<module>/start.sh` (not `demo/start.sh`)
- [ ] `docs/snap-demo-script.md` path references updated

## Implementation Notes

### `start.sh` REPO_ROOT fix

Current (3 levels up from `domains/snap/demo/`):
```bash
REPO_ROOT="$(cd "$SCRIPT_DIR/../../.." && pwd)"
```

New (4 levels up from `domains/snap/output/demo-eligibility/`):
```bash
REPO_ROOT="$(cd "$SCRIPT_DIR/../../../.." && pwd)"
```

The `cd "$SCRIPT_DIR"` before uvicorn and the relative `static/` mount in `main.py` both remain correct — they are relative to the script's own directory.

### `x.py` resolve_paths patch

```python
# Before
"demo_sh":  base / "demo" / "start.sh",
"demo_req": base / "demo" / "requirements.txt",

# After
"demo_sh":  base / "output" / f"demo-{module}" / "start.sh",
"demo_req": base / "output" / f"demo-{module}" / "requirements.txt",
```
