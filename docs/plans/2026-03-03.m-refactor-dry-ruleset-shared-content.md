# refactor: extract shared content from ruleset commands into core/ruleset-shared.md

## Overview

`extract-ruleset.md` and `update-ruleset.md` contain ~125 lines of word-for-word identical
content. Any change to shared sections (scoring rubric, pre-flight logic, shared procedures)
requires editing two files and risks drift. Extract shared content into `core/ruleset-shared.md`;
both commands read it via a directive.

Token cost is neutral: shared lines are removed from each command and loaded via a read directive,
so per-invocation line counts stay approximately the same.

---

## Implementation Steps

### Step 1 — Create `core/ruleset-shared.md`

New file containing all shared content. Sections in order:

1. **Pre-flight Checks 3–6** — Input docs, filename valid, ai-guidance, multi-doc
2. **Scoring Rubric** — full table and special cases
3. **CIVIL Reference** — compact version (Do NOT read tools/ + read authoring-constraints directive)
4. **Shared Procedures** — Sub-A (Validate), Sub-B (Generate Computation Graph), Sub-C (Footer)
5. **Common Mistakes to Avoid** — full bullet list

Add a header comment explaining the file's purpose:
```markdown
# Ruleset Command — Shared Content

Shared by `/extract-ruleset` and `/update-ruleset`. Read via directive at the start of each command.
Do not invoke this file directly.
```

**Exact content to use** (copy verbatim from `extract-ruleset.md`):

- Pre-flight checks 3–6: lines 32–93
- Scoring Rubric section: lines 97–112
- CIVIL Reference section: lines 115–122
- Shared Procedures section: lines 480–531
- Common Mistakes section: lines 551–560

The shared file uses the heading `### Pre-flight Checks 3–6` internally.
Command files reference it as "run shared pre-flight checks 3–6 from `core/ruleset-shared.md`".

## Dependencies & Risks

- **No tool changes** — documentation refactor only
- **Main risk:** step references to "Scoring Rubric above" or "Sub-A" in process steps still work
  after the sections are moved. Mitigation: grep for all `Sub-A`, `Sub-B`, `Sub-C`, `Scoring Rubric`
  references in both command files and verify each one still resolves correctly post-move.
- **Read directive ordering:** Claude must read `ruleset-shared.md` before executing pre-flight.
  The directive appears before `## Pre-flight` to enforce this.
