# refactor: Slim extract-ruleset Pre-flight to Two Checks

## Overview

After the command decomposition (PR #10), `extract-ruleset.md` pre-flight has two checks that no longer belong there:

- **OPA available?** — extract-ruleset never calls OPA post-decomposition; this is just an early warning. `/transpile-and-test` already has a proper OPA check with fallback behavior.
- **Makefile target exists?** — scaffolding the Makefile is infrastructure setup for `make` invocations, which only `/transpile-and-test` makes. Moving the scaffold there keeps responsibility with the command that actually needs it.

After this change, `extract-ruleset` pre-flight has exactly two checks that are genuinely its concern:
1. Domain folder exists?
2. Input docs present?

## Proposed Solution

### `extract-ruleset.md` — remove items 3 and 4 from pre-flight

Delete pre-flight item 3 (OPA available) and item 4 (Makefile target exists) in their entirety.

Result:
```
## Pre-flight

1. Domain folder exists?
2. Input docs present?
```

### `transpile-and-test.md` — upgrade Makefile check from "error" to "scaffold and continue"

Currently, `/transpile-and-test` pre-flight item 2 errors when the Makefile target is missing:
> "No Makefile target found. Run `/extract-ruleset <domain>` first to scaffold it." Stop.

Change this to scaffold the target (same template as the block in extract-ruleset) and continue:
- If CIVIL file exists: derive `<DOMAIN>_PACKAGE` from its `module:` field
- If CIVIL file does not exist: use `<domain>.<program>` as the package placeholder

The full Makefile template moves verbatim from `extract-ruleset.md` pre-flight item 4 into `transpile-and-test.md` pre-flight item 2.

## Acceptance Criteria

- [x] `extract-ruleset.md` pre-flight contains exactly 2 items (domain folder + input docs)
- [x] `extract-ruleset.md` has no OPA check and no Makefile scaffold logic
- [x] `transpile-and-test.md` Makefile check scaffolds-and-continues instead of erroring
- [x] `transpile-and-test.md` carries the full Makefile block template (all targets, variables, .PHONY)
- [x] `transpile-and-test.md` derives `<DOMAIN>_PACKAGE` from CIVIL `module:` field if CIVIL exists, falls back to `<domain>.<program>`
